# 性能基线规范

关键模块首次通过评审时，记录性能基线。后续改动不允许无理由的性能劣化。

## 基线目录

每个项目在根目录下建 `benchmarks/` 目录存放基线文件。

## 基线文件格式

```json
{
  "module": "module_name",
  "created": "YYYY-MM-DD",
  "updated": "YYYY-MM-DD",
  "metrics": {
    "time_ms": { "value": 45, "resolution": "1024x1024" },
    "memory_mb": { "value": 128, "resolution": "1024x1024" },
    "gpu_memory_mb": { "value": 64, "resolution": "1024x1024" }
  }
}
```

## 双重标准：绝对阈值 + 相对基线

| 标准 | 说明 | 适用时机 |
|------|------|---------|
| **绝对阈值** | 在 Spec 验收断言中预定义的性能上限（PERF: xxx < Nms） | 首次必须达标 |
| **相对基线** | 后续改动不劣化超过 10% | 后续每次评审 |

首次基线录入时，如果未达到 Spec 定义的绝对阈值，评审不通过。

## 基线规则

| 规则 | 说明 |
|------|------|
| **首次记录** | 关键模块首次通过评审时自动记录基线 |
| **绝对达标** | 首次基线必须满足 Spec 中的 PERF 绝对阈值 |
| **劣化阈值** | 后续改动耗时或内存增长超过 10% 即判定不通过 |
| **合理劣化** | 因功能增加导致的合理劣化，需在评审报告中说明原因并更新基线 |
| **优化记录** | 性能优化后更新基线，记录优化幅度 |
