# Never Again 规则引擎

每当 Agent 在评审或人工验收中犯错，**立即沉淀为一条永久规则**，并注册为可执行 Skill，确保同类错误在整个项目生命周期内不再发生。

## 机制流程

```
Agent 犯错（评审扣分 / 人工反馈）
    │
    ▼
┌─────────────────────────────────────┐
│  Step 1：提炼规则                    │
│  从具体错误中抽象出通用规则          │
│  写入 rules/RULE-XXX.md             │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  Step 2：注册为 Skill               │
│  将规则转化为可自动执行的检查逻辑    │
│  注册到 Agent 的 Skill 列表中        │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  Step 3：纳入评审门禁               │
│  后续所有评审自动加载全部 Skill 检查 │
│  确保 Never Again                   │
└─────────────────────────────────────┘
```

## 规则文件格式（rules/RULE-XXX.md）

```markdown
# RULE-XXX: [规则名称]

## 触发场景
[什么情况下会触发这个错误]

## 错误案例
[首次犯错的具体代码/行为]

## 正确做法
[应该怎么做]

## 检查方法
[自动化检查的具体步骤/命令/代码模式匹配]

## 来源
- Session: [哪个 Session 犯的错]
- 评审轮次: 第 N 轮
- 评审维度: #X [维度名称]
- 日期: YYYY-MM-DD
```

## Skill 注册规范

每条规则对应一个 Skill，Agent 在编码和评审时自动调用：

| 属性 | 说明 |
|------|------|
| **Skill 名称** | `check-rule-XXX`，与规则编号一致 |
| **触发时机** | 每次评审的第 0 步（在 12 项细则之前执行） |
| **执行方式** | 自动扫描本次提交的所有代码，检查是否违反该规则 |
| **失败处理** | 任何一条规则检查未通过，直接阻断评审，先修复再进入正式评审 |
| **累积效应** | 规则只增不删，随着项目推进，Agent 的防错能力持续增强 |

## 运作规则

1. **即时沉淀**：犯错后当场写规则，不延迟、不批量处理
2. **抽象通用**：规则必须从具体错误中抽象出通用模式，而非仅针对个案
3. **可自动检查**：每条规则必须包含可执行的检查方法（grep 模式 / AST 扫描 / 测试用例）
4. **前置拦截**：规则检查在 12 项评审之前执行，不合格直接打回，不浪费评审时间
5. **只增不删**：规则库只能新增和修订，不能删除，确保历史教训永不丢失
6. **人工可追加**：人工验收中发现的问题同样走此流程，由 Agent 编写规则并注册 Skill
