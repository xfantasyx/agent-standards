# 编码前设计评审（Design Review）

复杂任务在编码前，Agent 必须先输出设计方案，经人工确认后再编码。避免方向错误导致整轮返工。

## 触发条件

以下任一条件满足即为"复杂任务"，必须先出设计方案：

- 预估编码时间 ≥ 3h
- 涉及新算法实现
- 涉及跨模块接口设计
- 涉及 GPU Compute Shader
- 涉及与外部格式/引擎的集成

简单任务（< 3h、接口明确、算法成熟）可跳过此步骤，直接编码。

## 设计方案格式

```
┌─────────────────────────────────────────┐
│ 设计方案 — [Session 名称]               │
├─────────────────────────────────────────┤
│ 目标：一句话说明要实现什么               │
│ 输入：接收什么数据/接口                  │
│ 输出：产出什么数据/接口                  │
│ 算法选型：用什么算法，为什么选它         │
│ 数据流：数据怎么流转（伪代码或图）       │
│ 接口定义：对外暴露的函数签名             │
│ 风险点：预判可能出问题的地方             │
│ 备选方案：如果主方案不行，Plan B 是什么  │
└─────────────────────────────────────────┘
```

## 流程

```
Agent 输出设计方案
    │
    ▼
人工审阅 → 确认 / 修改方向 / 否决换方案
    │
    ▼ (确认后)
Agent 开始编码
```
