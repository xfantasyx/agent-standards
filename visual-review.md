# 视觉质量评审协议（Visual Review Protocol）

自动化测试对视觉效果无能为力。对于"看起来对不对"类任务，需要专门的视觉评审机制。

## 缺陷 #4 修复：视觉任务无法自动化测试

### 视觉任务识别

以下类型任务属于视觉质量任务：

- 侵蚀效果（沟壑/崩塌是否自然）
- 悬崖贴合（接缝是否可见）
- 河流路径（是否合理）
- 纹理过渡（Triplanar 接缝）
- 散布效果（碎石/岩石分布）
- 整体地形外观

### 视觉评审流程

```
Agent 编码完成（通过自动化评审）
    │
    ▼
┌─────────────────────────────────────┐
│  Agent 生成截图集                    │
│  多角度、多参数组合                  │
│  输出到 screenshots/T-XXX/           │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  Agent 自评视觉效果                  │
│  逐项对照视觉验收清单               │
│  标注可能有问题的区域                │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  人工视觉验收                        │
│  查看截图 + 实时运行                 │
│  反馈具体问题区域和修改方向          │
└─────────────────────────────────────┘
```

### 截图要求

每个视觉任务至少生成以下截图：

| 截图类型 | 数量 | 说明 |
|----------|------|------|
| **全景** | 2-3 张 | 不同角度的整体效果 |
| **特写** | 3-5 张 | 关键细节区域（接缝、过渡带等） |
| **对比** | 2 张 | 处理前 vs 处理后 |
| **极端参数** | 2-3 张 | 最大/最小参数组合，验证不崩溃 |
| **线框叠加** | 1 张 | 网格 + 渲染叠加，检查几何 |

### 视觉验收清单（通用）

| # | 检查项 | 标准 |
|---|--------|------|
| V1 | **自然度** | 不出现明显的程序化痕迹（重复纹理、网格状分布） |
| V2 | **接缝** | 模块间过渡平滑，无可见接缝 |
| V3 | **比例** | 地貌特征比例合理（悬崖不会太薄/太厚） |
| V4 | **连续性** | 地形连续，无不合理的断裂或突变 |
| V5 | **极端情况** | 极端参数下不出现视觉异常（穿模/黑面/NaN） |

### 数值辅助视觉判断

对于可量化的视觉指标，建立数值检查作为辅助：

| 指标 | 自动检查方式 |
|------|------------|
| 接缝宽度 | 相邻模块边界高度差 < 阈值 |
| 法线连续性 | 接缝处法线角度差 < 阈值 |
| 分布均匀度 | 散布点密度方差 < 阈值 |
| 无穿模 | 模型底部不低于地形表面 |
| 无 NaN | 输出 buffer 无 NaN/Inf |

这些数值检查可以纳入自动化测试，作为视觉质量的底线保障。
