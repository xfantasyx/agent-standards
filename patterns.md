# 模式库（Patterns Library）

项目中确立的编码模式集中记录。Agent 编码时先查模式库，不用每次重新做设计决策。

> 与 Never Again 互补：模式库 = "应该怎么做"，Never Again = "不该怎么做"

## 使用规则

| 规则 | 说明 |
|------|------|
| **编码前查阅** | 开始编码前检查模式库中是否有适用的模式 |
| **严格遵循** | 已确立的模式必须遵循，不允许同一问题用不同方案 |
| **新模式提案** | 遇到新的设计决策时，编码完成后将决策沉淀为新模式 |
| **模式修订** | 发现更优方案时，更新模式而非绕过，确保全项目一致 |
| **项目专属** | 每个项目在根目录下维护自己的 `patterns.md` |

## 模式记录格式

每条模式按以下格式记录：

```markdown
### P-XXX: [模式名称]

**适用场景**：[什么时候用这个模式]

**模式**：
[代码示例或伪代码]

**为什么这样做**：[选择这个方案的理由]

**反模式**：[不应该怎么做]
```

## 模式示例

```markdown
### P-001: GPU 资源生命周期管理

**适用场景**：任何创建 GPU Buffer/Texture/Pipeline 的代码

**模式**：
使用 RAII 包装器或 try-finally 确保资源释放：
const buffer = device.createBuffer(...)
try {
  // 使用 buffer
} finally {
  buffer.destroy()
}

**为什么这样做**：中间抛异常时资源不会泄漏

**反模式**：
在函数末尾调用 destroy()，中间无异常保护
```

## 模式与其他机制的关系

```
编码前：
  查 patterns.md → 知道"应该怎么做"
  查 rules/ → 知道"不该怎么做"

编码后：
  新的设计决策 → 沉淀到 patterns.md
  新的错误教训 → 沉淀到 rules/
```
