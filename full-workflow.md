# Agent 编码完整流程

从任务开始到完成的端到端流程。

## 完整流程图

```
┌──────────────────────────────────────────────────────┐
│  Phase 0: 设计评审（复杂任务必须，简单任务可跳过）     │
│  Agent 输出设计方案 → 人工确认方向                     │
│  参见：design-review.md                               │
└──────────────────┬───────────────────────────────────┘
                   │ 方向确认
                   ▼
┌──────────────────────────────────────────────────────┐
│  Phase 1: Agent 编码                                  │
│  按设计方案实现功能                                    │
└──────────────────┬───────────────────────────────────┘
                   │ 编码完成
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 0: Never Again 规则全量扫描                     │
│  执行所有已注册的 check-rule-XXX                      │
│  任何规则未通过 → 直接打回修复                         │
│  参见：never-again.md                                 │
└──────────────────┬───────────────────────────────────┘
                   │ 全部通过
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 1: 12 项质量评审                                │
│  逐项打分，必须 120/120                               │
│  参见：quality-gate.md                                │
└──────────────────┬───────────────────────────────────┘
                   │ 满分通过
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 2: 回归验证                                     │
│  跑全量已有测试，确保不破坏已有模块                    │
│  任何测试失败 → 打回修复                               │
│  参见：regression.md                                  │
└──────────────────┬───────────────────────────────────┘
                   │ 全部通过
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 3: 性能基线对比                                  │
│  对比关键模块的性能基线                                │
│  劣化超过 10% → 打回优化                               │
│  参见：performance-baseline.md                        │
└──────────────────┬───────────────────────────────────┘
                   │ 达标
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 4: 人工验收                                     │
│  运行、视觉检查、性能测量                             │
│  发现问题 → Agent 修复 + 写 Never Again 规则           │
└──────────────────────────────────────────────────────┘
```

## 失败处理

- **任何步骤失败**：修复后从 Step 0 重新开始全量评审
- **评审中发现新错误模式**：立即写 Never Again 规则
- **人工验收不通过**：Agent 修复 + 补充 Never Again 规则 + 重新走全流程

## Session 上下文传递

每个 Agent Session 启动时读取、结束时更新 `STATUS.md`：

### 启动时必读

| 文件 | 作用 |
|------|------|
| **CLAUDE.md** | 规则、标准、项目架构 |
| **STATUS.md** | 当前进度、已完成模块、关键接口、已知问题 |

### 结束时必须更新 STATUS.md

| 更新项 | 说明 |
|--------|------|
| 当前进度 | 正在做什么、下一个任务是什么 |
| 已完成模块 | 模块名 + 入口文件 + 评审通过日期 |
| 关键接口 | 对外暴露的核心函数签名 |
| 已知问题 | 未解决的 bug 或技术债 |
| Never Again 规则数 | 当前累计数量 |
| 性能基线摘要 | 各模块基线数据 |
