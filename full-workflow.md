# Agent 编码完整流程

从任务开始到完成的端到端流程。

## 完整流程图

```
┌──────────────────────────────────────────────────────┐
│  Phase 0: 设计评审（复杂任务必须，简单任务可跳过）     │
│  Agent 输出设计方案 → 人工确认方向                     │
│  参见：design-review.md                               │
└──────────────────┬───────────────────────────────────┘
                   │ 方向确认
                   ▼
┌──────────────────────────────────────────────────────┐
│  Phase 1: Spec 生成（定义接口的任务必须）              │
│  设计方案 → 编码契约（接口签名+边界约束+验收断言）     │
│  计算 source_hash，解析 exports/imports               │
│  人工确认 Spec → 任务状态 → spec_ready                │
│  参见：spec-layer.md                                  │
└──────────────────┬───────────────────────────────────┘
                   │ Spec 确认
                   ▼
┌──────────────────────────────────────────────────────┐
│  Phase 2: 测试先行（TDD）                             │
│  按 Spec 验收断言编写 3-5 个测试 + 边界条件测试        │
│  测试能编译但全部 FAIL → 人工确认测试合理性            │
│  参见：test-first.md                                  │
└──────────────────┬───────────────────────────────────┘
                   │ 测试确认
                   ▼
┌──────────────────────────────────────────────────────┐
│  Phase 3: Agent 编码（增量检查点 + 自检门）            │
│  查 patterns.md → 按模式编码                          │
│  查 rules/ → 避开已知陷阱                             │
│  每完成一个核心函数 → 自检门(H/I/G) + 编译 + 跑测试   │
│  参见：incremental-checkpoint.md, self-check-gates.md │
└──────────────────┬───────────────────────────────────┘
                   │ 全部测试 PASS + 自检门通过
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 0: Never Again 规则全量扫描                     │
│  执行所有已注册的 check-rule-XXX                      │
│  任何规则未通过 → 直接打回修复                         │
│  参见：never-again.md                                 │
└──────────────────┬───────────────────────────────────┘
                   │ 全部通过
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 1: 自动化预检（有脚本时执行）                    │
│  spec_verify.py → Spec ↔ 代码接口一致性               │
│  auto_review.py → 可自动化评审项预检                   │
│  参见：automated-verification.md                      │
└──────────────────┬───────────────────────────────────┘
                   │ 全部通过
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 2: 13 项质量评审                                │
│  逐项打分，必须 130/130                               │
│  含 #13 Spec 一致性与跨任务兼容性                     │
│  参见：quality-gate.md                                │
└──────────────────┬───────────────────────────────────┘
                   │ 满分通过
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 3: 回归验证                                     │
│  跑全量已有测试，确保不破坏已有模块                    │
│  任何测试失败 → 打回修复                               │
│  参见：regression.md                                  │
└──────────────────┬───────────────────────────────────┘
                   │ 全部通过
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 4: 性能基线对比                                  │
│  对比关键模块的性能基线                                │
│  劣化超过 10% → 打回优化                               │
│  参见：performance-baseline.md                        │
└──────────────────┬───────────────────────────────────┘
                   │ 达标
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 5: 人工验收                                     │
│  运行、视觉检查、性能测量                             │
│  发现问题 → Agent 修复 + 写 Never Again 规则           │
└──────────────────┬───────────────────────────────────┘
                   │ 通过
                   ▼
┌──────────────────────────────────────────────────────┐
│  Step 6: 收尾                                         │
│  新设计决策 → 沉淀到 patterns.md                      │
│  更新 STATUS.md / TASKS.md                            │
│  任务状态 → done                                      │
└──────────────────────────────────────────────────────┘
```

## 任务状态生命周期

```
🔴 planned ──Spec生成──→ 📋 spec_ready ──编码──→ 🟡 in_progress ──评审通过──→ 🔵 done
```

## 失败处理

- **任何步骤失败**：修复后从 Step 0 重新开始全量评审
- **评审中发现新错误模式**：立即写 Never Again 规则
- **人工验收不通过**：Agent 修复 + 补充 Never Again 规则 + 重新走全流程
- **设计源文档变更**：检测 source_hash 漂移 → 更新 Spec → 必要时更新代码

## Session 上下文传递

每个 Agent Session 启动时读取、结束时更新项目状态文件：

### 启动时必读

| 文件 | 作用 |
|------|------|
| **CLAUDE.md** | 规则、标准、项目架构 |
| **STATUS.md** | 当前进度、已完成模块、关键接口、已知问题 |
| **TASKS.md** | 任务注册表（状态、依赖、接口） |

### 结束时必须更新

| 文件 | 更新内容 |
|------|---------|
| **STATUS.md** | 进度、已完成模块、接口、已知问题、规则数、基线 |
| **TASKS.md** | 任务状态、产出接口、评审轮次、关联规则 |
| **patterns.md** | 新的设计决策沉淀（如有） |
