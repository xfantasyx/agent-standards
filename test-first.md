# 测试先行（Test-Driven Development）

编码前先写测试用例，定义"什么叫对"。先明确目标，再动手实现。

## 流程

```
Agent 接到任务
    │
    ▼
┌─────────────────────────────────────┐
│  Step 1：编写测试用例                │
│  3-5 个关键测试 + 边界条件测试       │
│  测试必须能编译，执行后全部 FAIL     │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  Step 2：评审 Agent 确认测试合理性    │
│  测试是否覆盖核心功能？              │
│  边界条件是否充分？                  │
│  预期值是否正确？                    │
│  （L1 由编码 Agent 自行保障）        │
└──────────────┬──────────────────────┘
               │ 确认通过
               ▼
┌─────────────────────────────────────┐
│  Step 3：编码实现，让测试逐个 PASS   │
│  每实现一个函数，跑一次测试          │
│  全部 PASS 后进入评审流程            │
└─────────────────────────────────────┘
```

## 测试用例要求

| 要求 | 说明 |
|------|------|
| **数量** | 每个任务至少 3-5 个测试用例 |
| **覆盖** | 必须包含：正常路径、边界条件、异常输入 |
| **独立** | 每个测试可独立运行，不依赖执行顺序 |
| **确定性** | 相同输入必须产生相同结果，不允许随机性 |
| **命名** | 测试名称描述被测行为，如 `test_perlin_noise_range_is_minus1_to_1` |
| **快速** | 单个测试执行时间 < 1s，GPU 测试 < 5s |

## 边界条件清单（通用）

编写测试时检查以下边界条件：

- 零值 / 空值输入
- 最小 / 最大分辨率
- 单像素 / 单元格
- 全平 / 全陡地形
- 浮点精度极限（极小/极大值）
- 数组边界（首元素/末元素/越界）
