# 人工审核分级机制（Human Review Tiers）

防止人工成为流程瓶颈。按任务复杂度分级，不同级别需要不同程度的人工介入。

## 人工职责（适用于非技术人员）

人工不审技术细节，只负责两件事：

| 职责 | 说明 | 时机 |
|------|------|------|
| **方向确认** | 确认 Agent 要做的事是不是你想要的 | 设计评审时（L3）/ 任务启动时（L1/L2） |
| **效果验收** | 运行程序看效果对不对、截图看起来自然不自然 | 最终验收时 |

人工**不需要**：
- 审查接口签名是否正确
- 审查测试用例是否合理
- 审查 Spec 的边界约束
- 审查代码质量

这些全部由 Agent（编码 Agent + 评审 Agent）闭环保障。

---

## 三级审核制度

| 级别 | 适用任务 | 人工介入点 | 预计占比 |
|------|---------|-----------|---------|
| **L1 轻量** | 简单任务（<3h，接口明确，无新算法） | 方向确认（口头）+ 效果验收 | ~40% |
| **L2 标准** | 中等任务（有新接口，有依赖） | 方向确认 + 效果验收 | ~40% |
| **L3 重度** | 复杂任务（新算法，跨模块，GPU） | 方向确认（设计评审）+ 效果验收（含视觉评审截图） | ~20% |

### 各级别详细流程

#### L1 轻量（跳过设计评审和 Spec）

```
人工方向确认（口头） → Agent 编码 → 自检门 → 评审门 → 快速回归测试 → 人工效果验收
```

**人工介入**：
- 任务启动时：口头确认"要做的事对不对"即可
- 最终验收时：运行程序看效果

**L1 回归测试规则**：
- L1 跑"快速测试"：本模块单元测试 + 直接依赖的接口测试
- 时间预算：<30s
- 如果快速测试不存在（首个模块），跳过回归但必须在人工验收中说明

条件：
- 预估 <3h
- 接口已在上游 Spec 中定义
- 算法成熟（如噪声函数是标准实现）
- L1 任务不定义新的对外接口。如果任务需要定义新接口（新的公共函数/类/类型），自动升级为 L2。

#### L2 标准（跳过设计评审，Spec 由评审 Agent 审查）

```
人工方向确认 → Spec 生成 → 评审 Agent 审查 Spec → 测试先行 → Agent 编码 → 全流程评审 → 人工效果验收
```

**人工介入**：
- 任务启动时：确认"要做的事对不对"
- 最终验收时：运行程序看效果

条件：
- 定义新接口
- 有跨模块依赖
- 预估 3-6h

#### L3 重度（设计评审方向确认 + 效果验收）

```
设计评审 → 人工确认方向 → Spec → 评审 Agent 审查 → 测试 → 评审 Agent 确认测试合理性 → 编码 → 全流程评审 → 人工效果验收（含视觉评审截图）
```

**人工介入**：
- 设计评审时：确认"方向对不对"（不审技术方案细节）
- 最终验收时：运行程序看效果 + 看截图是否自然

技术质量（Spec 质量、测试合理性、代码质量）全部由评审 Agent 闭环保障。

条件：
- 新算法（侵蚀、贴合等）
- GPU Compute Shader
- 跨模块架构决策
- 预估 >6h

### 级别判定规则

Agent 在开始任务时自动判定级别，在评审报告开头声明：

```
审核级别：L2（标准）
理由：定义新接口 HeightmapGenerator，有上游依赖 T-007
```

人工可以在任何时候升级审核级别（但不能降级）。

### 批量确认

对于 L1 任务，人工可以批量验收：

- 累积 3-5 个 L1 任务后一次性验收
- 前提：每个任务都已独立通过评审门禁
- 验收时运行程序看效果，不需要检查代码或评审证据

## 视觉任务评审

自动化测试对视觉效果无能为力。对于"看起来对不对"类任务，需要专门的视觉评审机制。

### 视觉任务识别

以下类型任务属于视觉质量任务：

- 侵蚀效果（沟壑/崩塌是否自然）
- 悬崖贴合（接缝是否可见）
- 河流路径（是否合理）
- 纹理过渡（Triplanar 接缝）
- 散布效果（碎石/岩石分布）
- 整体地形外观

### 截图要求

每个视觉任务至少生成以下截图：

| 截图类型 | 数量 | 说明 |
|----------|------|------|
| **全景** | 2-3 张 | 不同角度的整体效果 |
| **特写** | 3-5 张 | 关键细节区域（接缝、过渡带等） |
| **对比** | 2 张 | 处理前 vs 处理后 |
| **极端参数** | 2-3 张 | 最大/最小参数组合，验证不崩溃 |
| **线框叠加** | 1 张 | 网格 + 渲染叠加，检查几何 |

### 视觉验收清单（V1-V5）

| # | 检查项 | 标准 |
|---|--------|------|
| V1 | **自然度** | 不出现明显的程序化痕迹（重复纹理、网格状分布） |
| V2 | **接缝** | 模块间过渡平滑，无可见接缝 |
| V3 | **比例** | 地貌特征比例合理（悬崖不会太薄/太厚） |
| V4 | **连续性** | 地形连续，无不合理的断裂或突变 |
| V5 | **极端情况** | 极端参数下不出现视觉异常（穿模/黑面/NaN） |

### 数值辅助视觉判断

对于可量化的视觉指标，建立数值检查作为辅助：

| 指标 | 自动检查方式 |
|------|------------|
| 接缝宽度 | 相邻模块边界高度差 < 阈值 |
| 法线连续性 | 接缝处法线角度差 < 阈值 |
| 分布均匀度 | 散布点密度方差 < 阈值 |
| 无穿模 | 模型底部不低于地形表面 |
| 无 NaN | 输出 buffer 无 NaN/Inf |

这些数值检查可以纳入自动化测试，作为视觉质量的底线保障。
