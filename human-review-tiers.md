# 人工审核分级机制（Human Review Tiers）

防止人工成为流程瓶颈。按任务复杂度分级，不同级别需要不同程度的人工介入。

## 缺陷 #3 修复：人工瓶颈

### 三级审核制度

| 级别 | 适用任务 | 人工介入点 | 预计占比 |
|------|---------|-----------|---------|
| **L1 轻量** | 简单任务（<3h，接口明确，无新算法） | 仅最终验收 | ~40% |
| **L2 标准** | 中等任务（有新接口，有依赖） | Spec确认 + 最终验收 | ~40% |
| **L3 重度** | 复杂任务（新算法，跨模块，GPU） | 全流程人工介入 | ~20% |

### 各级别详细流程

#### L1 轻量（跳过设计评审和 Spec，跳过测试确认）

```
Agent 编码 → 自检门 → 评审门 → 回归 → 人工验收
```

条件：
- 预估 <3h
- 接口已在上游 Spec 中定义
- 算法成熟（如噪声函数是标准实现）

#### L2 标准（跳过设计评审，需要 Spec 确认）

```
Spec 生成 → 人工确认 Spec → 测试先行 → Agent 编码 → 全流程评审 → 人工验收
```

条件：
- 定义新接口
- 有跨模块依赖
- 预估 3-6h

#### L3 重度（全流程人工介入）

```
设计评审 → 人工确认 → Spec → 人工确认 → 测试 → 人工确认 → 编码 → 全流程评审 → 人工验收
```

条件：
- 新算法（侵蚀、贴合等）
- GPU Compute Shader
- 跨模块架构决策
- 预估 >6h

### 级别判定规则

Agent 在开始任务时自动判定级别，在评审报告开头声明：

```
审核级别：L2（标准）
理由：定义新接口 HeightmapGenerator，有上游依赖 T-007
```

人工可以在任何时候升级审核级别（但不能降级）。

### 批量确认

对于 L1 任务，人工可以批量验收：

- 累积 3-5 个 L1 任务后一次性验收
- 前提：每个任务都已独立通过评审门禁
- 验收时随机深入检查 1-2 个任务的评审证据

## 视觉任务评审

自动化测试对视觉效果无能为力。对于"看起来对不对"类任务，需要专门的视觉评审机制。

### 视觉任务识别

以下类型任务属于视觉质量任务：

- 侵蚀效果（沟壑/崩塌是否自然）
- 悬崖贴合（接缝是否可见）
- 河流路径（是否合理）
- 纹理过渡（Triplanar 接缝）
- 散布效果（碎石/岩石分布）
- 整体地形外观

### 截图要求

每个视觉任务至少生成以下截图：

| 截图类型 | 数量 | 说明 |
|----------|------|------|
| **全景** | 2-3 张 | 不同角度的整体效果 |
| **特写** | 3-5 张 | 关键细节区域（接缝、过渡带等） |
| **对比** | 2 张 | 处理前 vs 处理后 |
| **极端参数** | 2-3 张 | 最大/最小参数组合，验证不崩溃 |
| **线框叠加** | 1 张 | 网格 + 渲染叠加，检查几何 |

### 视觉验收清单（V1-V5）

| # | 检查项 | 标准 |
|---|--------|------|
| V1 | **自然度** | 不出现明显的程序化痕迹（重复纹理、网格状分布） |
| V2 | **接缝** | 模块间过渡平滑，无可见接缝 |
| V3 | **比例** | 地貌特征比例合理（悬崖不会太薄/太厚） |
| V4 | **连续性** | 地形连续，无不合理的断裂或突变 |
| V5 | **极端情况** | 极端参数下不出现视觉异常（穿模/黑面/NaN） |

### 数值辅助视觉判断

对于可量化的视觉指标，建立数值检查作为辅助：

| 指标 | 自动检查方式 |
|------|------------|
| 接缝宽度 | 相邻模块边界高度差 < 阈值 |
| 法线连续性 | 接缝处法线角度差 < 阈值 |
| 分布均匀度 | 散布点密度方差 < 阈值 |
| 无穿模 | 模型底部不低于地形表面 |
| 无 NaN | 输出 buffer 无 NaN/Inf |

这些数值检查可以纳入自动化测试，作为视觉质量的底线保障。
