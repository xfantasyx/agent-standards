# 自检门（Self-Check Gates）

编码过程中的前置检查，在正式评审之前拦截低级错误。和增量检查点配合使用——检查点是"何时检查"，自检门是"检查什么"。

## 头文件 / 接口自检门

| ID | 检查项 | 要求 |
|----|--------|------|
| H1 | **文件自包含** | 每个头文件/模块入口包含所有需要的依赖，可独立编译 |
| H2 | **边界约束遵从** | 逐条对照 Spec 的 MUST / MUST NOT，每条都有对应实现 |
| H3 | **命名空间与导出** | 命名空间与 Spec 一致，导出符号与 exports 列表一致 |
| H4 | **接口签名匹配** | 实际接口签名与 Spec 中的接口定义逐字匹配 |

## 实现自检门

| ID | 检查项 | 要求 |
|----|--------|------|
| I1 | **平台假设** | 不假设特定平台行为；平台相关代码有条件编译 |
| I2 | **算法正确性** | 至少一个手工验证用例，核心公式与论文/参考一致 |
| I3 | **线程安全** | 共享可变状态有锁保护；回调接口标注线程安全级别 |
| I4 | **编码规范** | 遵循项目 patterns.md 中确立的模式 |
| I5 | **资源生命周期** | GPU/文件/网络资源用 RAII 或 try-finally 管理 |

## GPU / Shader 自检门

| ID | 检查项 | 要求 |
|----|--------|------|
| G1 | **Workgroup 尺寸** | Compute Shader workgroup 尺寸合理（通常 64/128/256） |
| G2 | **Buffer 越界** | 所有 buffer 访问有边界检查或保证不越界 |
| G3 | **同步** | 需要 barrier 的地方有 barrier，shared memory 无竞争 |
| G4 | **精度** | 浮点计算精度符合要求，中间结果不溢出 |

## 使用方式

```
编写一个函数/模块后：

1. 逐条过自检门清单（H1-H4, I1-I5, G1-G4 按适用性）
2. 任何一项不满足 → 立即修复
3. 全部满足 → 继续写下一个函数

注意：自检门是编码过程中持续执行的，不是写完后才检查
```

## 与评审门的关系

```
编码中 → 自检门（H/I/G，逐函数执行，拦截低级错误）
    │
    ▼
编码完成 → 评审门（12 项质量评审，全局维度打分）
```

自检门通过不代表评审一定通过，但能提前拦截约 60% 的评审 Critical 问题。
